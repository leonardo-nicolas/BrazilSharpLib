Documentation of Brazil #

Preface

With so much information we typed, something we always end up typing wrong, accidentally. And when it comes to documents (for example), there can be no different information, such as wrong document numbers (such as too many numbers or missing), words without their proper accents, etc. Because this can cause inconsistency when signing a contract and recognizing a notary firm (for example).

Thinking about it, the author thought about this project and put it into practice, with the main intention of making available portfolio in his GitHub repository to have what to show to recruiters and also took the opportunity to make it available to the public to import it in their projects (Whether cloning repository or importing via NuGET package).

After all, if you can with the end user avoid some problem by typo, such as sending an NF-e with the CNPJ of the issuing company, invalid, because of 1 digit more that accidentally left in the rush to type, for example, is to be able to avoid worse future disorders.

That's the point! In addition to this, it will also have more methods/functions (In addition to the method that calculates the 13th), for labor calculations and help those who work with Personal Department or Human Resources. Just keep the library updated in the latest version always!

Below are the namespaces,classes, structures, interfaces, enum, delegates, methods, properties, fields or params and their descriptions:

NOTE: The colors defined above serve to better identify who is who...

  • BrazilSharp: The main namespace.
  • Brazilsharp.Validate: A static class with a set of static methods for validating documents or other data. Below are the methods of this class, how to use and sample codes.
  • Brazilsharp.Utilities: A static class with a set of static methods to be used in some diverse situations.
  • Brazilsharp.BrazilianStates: An enumerator with all states of all Brazilian territory and its value is the ibge state code.
  • BrazilSharp.Calculations: Namespace with a set of classes for performing various calculations.
  • BrazilSharp.Calculations.WorkersCalculation: A static class with a set of static methods for Human Resources (or even Personnel Department) calculations.
  • BrazilSharp.Calculations.PeriodType: An enumerator for the aid of the BrazilSharp.Calculations.WorkersCalculation.Thirteenth method, at the time of using it to do 13th proportional calculation.
  • BrazilSharp.Exceptions: The namespace with all exception classes (inherited System.Exception class).
  • BrazilSharp.Exceptions.StateNotFoundException: The error class that will be used by some methods where the value is different from the values of the BrazilSharp.BrazilianStates enum.

Validations

Document and other data validation methods are found in the Validate class (within the BrazilSharpnamespace) and are static methods.

Its use is only to import the BrazilSharp namespace into the file in which the methods will be used (or use the global using in the case of DotNet 6 with C# 10, both onwards) and call by the method desired directly by the class, without instanting it; because the Validate class is a static class.

 

CPF

This method is used to validate whether any information in the parameter is a true CPF or not.

bool BrazilSharp.Validate.CPF(object? Expression) => validates any data in Expression, to determine whether it's a valid or invalid CPF. The Expression parameter accepts any type of object that is going to enter the argument, including Arrays. 

// Code in C#
using System;
using BrazilSharp;

namespace  ConsoleApp1 
{
    public class Program
    {
        public static void Main(string[] args)
        {
            Console.WriteLine("Enter with brazilian's CPF to validate it...");
            string cpf = Console.ReadLine();
            bool IsCpfValid = Validate.CPF(cpf);
            
            if (IsCpfValid)
            {
                Console.WriteLine("OK, It's right!");
            }
            else
            {
                Console.WriteLine("The CPF {0}, isn't right! This CPF is invalid!", cpf);
            }
        }
    }
}

FindStateVoterTitle

This method is used to "reveal" the state in which a voter id was issued.

Brazilsharp.BrazilianStates? Brazilsharp.Utilities.FindStateVoterTitle(object? Expression) => validates any data in Expression, to determine whether it's a valid voter registration number or not. Being valid, returns some value from BrazilSharp.Brazilianstates. Otherwise, returns null. The Expression parameter accepts any type of object that is going to enter the argument, including Arrays. 

// Code in C#
using System;
using BrazilSharp;

namespace  ConsoleApp1 
{
    public class Program
    {
        public static void Main(string[] args)
        {
            Console.WriteLine("Type a brazilian Voter Title to know where it was issued.");
            string titulo = Console.ReadLine();
            BrazilianStates? estadoEmissor = Utilities.FindStateVoterTitle(titulo);
            
            if (estadoEmissor != null)
            {
                string SiglaEstadoEmissor = Enum.GetName(typeof(BrazilianStates), estadoEmissor);
                Console.WriteLine("Alright, it's OK! This voter title was issued at {0}!", SiglaEstadoEmissor);
            }
            else
            {
                Console.WriteLine("The Voter Title {0}, isn't valid and it's impossible to know where it was issued!", titulo);
            }
        }
    }
}

Thirteenth

This method is used to validate whether any information in the parameter is a true RENAVAM number or not.

double BrazilSharp.Calculations.WorkersCalculation.Thirteenth( double calculationBasis, bool EarlyWarning, BrazilSharp.Calculations.PeriodType Regime, short HowLongWorked) => Calculates the 13th proportional of an employee. The return of this method is double, with the 13th calculated. Below are the details and instructions of the parameters of this method:

Parameter Type Description
calculationBasis double The basis of calculation for the 13th...
EarlyWarning bool Set true if the employee is on notice. If not, false!
Regime BrazilSharp.Calculations.PeriodType

Will the calculation be done upon DAYS or MONTHS?

Report BrazilSharp.Calculations.PeriodType.Day to calculate in DAYS

Report BrazilSharp.Calculations.PeriodType.Month to calculate in MONTHS
HowLongWorked short

If the calculation is done upon DAYS, inform between 1 and 31, how many days the employee worked!

If the calculation is done over MONTHS, inform between 1 and 12, how many months the contractor worked!

If you are outside the range mentioned above, an error of type System.ArgumentOutOfRangeException will be trown!

 
// Code in C#
using System;
using BrazilSharp;
using BrazilSharp.Calculations;

namespace  ConsoleApp1 
{
    public class Program
    {
        public static void Main(string[] args)
        {
            Console.WriteLine("Let's Calculate the 13th Proportional");

            Console.WriteLine("Enter the employee's name");
            string nome = Console.ReadLine();
            
            Console.WriteLine("Inform the salary + night pay + unhealthy work + other gains provided for in Brazilian labor laws...");
            double BaseDeCalculo = Convert.ToDouble(Console.ReadLine());
            
            Console.WriteLine("Early warning? Press 'Y' or 'N'.");
            char RespostaAvisoPrevio = Console.ReadKey();
            bool AvisoPrevio = RespostaAvisoPrevio == 'Y' || RespostaAvisoPrevio == 'y';

            Console.WriteLine("Enter 'Day' if the calculation will be done IN DAYS:");
            Console.WriteLine("OU");
            Console.WriteLine("Enter 'Month', if the calculation will be done IN MONTHS:");
            string RespostaRegimeDeCalculo = Console.ReadLine();
            PeriodType RegimeDeCalculo = Enum.Parse(typeof(PeriodType), RespostaRegimeDeCalculo, false);

            Console.WriteLine("Quantos \"{0}'s\" que {1} trabalhou?", RespostaRegimeDeCalculo, nome);
            short PeriodoTrabalhado = Convert.ToInt16(Console.ReadLine());

            double ResultadoDoCaculo = WorkersCalculation.Thirteenth(BaseDeCalculo, AvisoPrevio, RegimeDeCalculo, PeriodoTrabalhado);
            
            Console.WriteLine("The thirteenth proportional of {0} is {1}.", nome, ResultadoDoCalculo);
            Console.WriteLine("If you add the salary and the rest of the calculation base, it will result in {0}", nome);
            Console.WriteLine("Congratulations {0}! You have a lot of money!!!", nome);
        }
    }
}
