Documentação de Brazil#

Prefácio

Com tantas informações que digitamos, alguma coisa sempre acabamos digitando errado, acidentalmente. E, quando se trata de documentos (por exemplo), não pode haver nenhuma informação diferente, como números de documentos errados (como número a mais ou faltando), palavras sem seus devidos acentos, etc. Pois isso pode causar inconsistência na hora de assinar um contrato e reconhecer firma em cartório (por exemplo).

Pensando nisso, o autor pensou neste projeto e pôs em prática, com a principal intenção de disponibilizar portfólio em seu repositório do GitHub para ter o quê mostrar aos recrutadores e aproveitou também para disponibilizar ao público para importá-lo em seus projetos (Seja clonando repositório ou importando via pacote NuGET).

Afinal, se puder com que o usuário final evite algum problema por erro de digitação, como enviar uma NF-e com o CNPJ da empresa emissora, inválido, por causa de 1 dígito a mais que saiu acidentalmente na pressa de digitar, por exemplo, é poder evitar piores transtornos futuros.

Essa é a intenção! Além dessa, também terá mais métodos/funções (Além do método que calcula o 13º), para cálculos trabalhistas e ajudar aqueles que trabalha com Departamento Pessoal ou Recursos Humanos. Só manter a biblioteca atualizada na última versão sempre!

Abaixo os namespaces, classes, estruturas, interfaces, enum, delegates, métodos, propriedades, campos e suas descrições:

OBS.: As cores acima definidas, servem para identificar melhor quem é quem...

  • BrazilSharp: O namespace principal.
  • BrazilSharp.Validate: Uma classe estática com um conjunto de métodos estáticos para validação de documentos ou outros dados. Mais abaixo os métodos desta classe, modo de se usar e códigos de exemplo.
  • BrazilSharp.Utilities: Uma classe estática com um conjunto de métodos estáticos para ser utilizado em algumas situações diversas.
  • BrazilSharp.BrazilianStates: Um enumerador com todos os estados de todo território brasileiro e seu valor é o código do estado IBGE.
  • BrazilSharp.Calculations: Namespace com um conjunto de classes para realização de cálculos diversos.
  • BrazilSharp.Calculations.WorkersCalculation: Uma classe estática com um conjunto de métodos estáticos para cálculos de Recursos Humanos (ou até de Departamento Pessoal).
  • BrazilSharp.Calculations.PeriodType: Um enumerador para auxiliação do método BrazilSharp.Calculations.WorkersCalculation.Thirteenth, na hora de usá-lo para fazer cálculo de 13º proporcional.
  • BrazilSharp.Exceptions: O namespace com todas as classes de excessões (classes herdadas de System.Exception).
  • BrazilSharp.Exceptions.StateNotFoundException: A classe de erro que será usada por alguns métodos em que o valor for diferente dos valores do enum BrazilSharp.BrazilianStates.

Validações

Os métodos de validação de documentos e de outros dados, se encontram na classe Validate (dentro do namespace BrazilSharp) e são métodos estáticos.

Seu uso consiste apenas em importar o namespace BrazilSharp no arquivo em que se usará os métodos (Ou utilizar o global using no caso do DotNet 6 com o C# 10, ambos por diante) e chamar pelo método desejado diretamente pela classe, sem instanciá-la; uma vez que a classe Validate é uma classe estática.

 

CPF

Este método é utilizado para validar se alguma informação qualquer no parâmetro, é um CPF verdadeiro ou não.

bool BrazilSharp.Validate.CPF(object? Expression) => valida qualquer dado em Expression, pra determinar se é um CPF válido ou não. O parâmetro Expression Aceita qualquer tipo de objeto que for entrar no argumento, inclusive Arrays.

// Code in C#
using System;
using BrazilSharp;

namespace  ConsoleApp1 
{
    public class Program
    {
        public static void Main(string[] args)
        {
            Console.WriteLine("Entre com um CPF para validar...");
            string cpf = Console.ReadLine();
            bool IsCpfValid = Validate.CPF(cpf);
            
            if (IsCpfValid)
            {
                Console.WriteLine("OK, passou no teste! Este CPF é Válido!");
            }
            else
            {
                Console.WriteLine("O CPF {0}, não passou no teste! Este CPF é inválido!", cpf);
            }
        }
    }
}

Thirteenth

Este método é utilizado para validar se alguma informação qualquer no parâmetro, é um número de RENAVAM verdadeiro ou não.

double BrazilSharp.Calculations.WorkersCalculation.Thirteenth(double calculationBasis, bool EarlyWarning, BrazilSharp.Calculations.PeriodType Regime, short HowLongWorked) => Calcula o 13º proporcional de um empregado. O retorno deste método é Double, com o 13º calculado. Abaixo os detalhes e instruções dos parâmetros deste método:

Nome do parâmetro Tipo Descrição
calculationBasis double A base de cálculo para o 13º...
EarlyWarning bool Defina true se o empregado estiver de aviso prévio. Se não for o caso, false!
Regime BrazilSharp.Calculations.PeriodType

O calculo será feito em cima de DIAS ou MESES?

Informe BrazilSharp.Calculations.PeriodType.Day para calcular em DIAS

Informe BrazilSharp.Calculations.PeriodType.Month para calcular em MESES
HowLongWorked short

Se o calculo for feito em cima de DIAS, informe entre 1 e 31, quantos dias o empregado trabalhou!

Se o calculo for feito em cima de MESES, informe entre 1 e 12, quantos meses o empredado trabalhou!

Caso esteja fora do intervalo citado acima, um erro do tipo System.ArgumentOutOfRangeException será lançado!

 
// Code in C#
using System;
using BrazilSharp;
using BrazilSharp.Calculations;

namespace  ConsoleApp1 
{
    public class Program
    {
        public static void Main(string[] args)
        {
            Console.WriteLine("Vamos Calcular o 13º Proporcional");

            Console.WriteLine("Digite o nome do empregado");
            string nome = Console.ReadLine();
            
            Console.WriteLine("Informe o salário + adicional noturno + insalubridade + outros ganhos previstos em lei...");
            double BaseDeCalculo = Convert.ToDouble(Console.ReadLine());
            
            Console.WriteLine("Aviso Prévio? Tecle 'S' ou 'N'.");
            char RespostaAvisoPrevio = Console.ReadKey();
            bool AvisoPrevio = RespostaAvisoPrevio == 'S' || RespostaAvisoPrevio == 's';

            Console.WriteLine("Informe 'Day' se o cálculo será feito EM DIAS:");
            Console.WriteLine("OU");
            Console.WriteLine("Informe 'Month' se o calculo será feito EM MESES:");
            string RespostaRegimeDeCalculo = Console.ReadLine();
            PeriodType RegimeDeCalculo = Enum.Parse(typeof(PeriodType), RespostaRegimeDeCalculo, false);

            Console.WriteLine("Quantos \"{0}'s\" que {1} trabalhou?", RespostaRegimeDeCalculo, nome);
            short PeriodoTrabalhado = Convert.ToInt16(Console.ReadLine());

            double ResultadoDoCaculo = WorkersCalculation.Thirteenth(BaseDeCalculo, AvisoPrevio, RegimeDeCalculo, PeriodoTrabalhado);
            
            Console.WriteLine("O décimo terceiro de {0} é de {1}.", nome, ResultadoDoCalculo);
            Console.WriteLine("Se somar ao salário e o restante da base de cálculo, vai dar {0}", nome);
            Console.WriteLine("Parabéns {0}! Tá com muita da grana em!!!", nome);
            Console.WriteLine("Me dá uma garoupa ou um lobo-guará? Brincadeira! :D :D");
        }
    }
}
